# 背景

我们希望可以让PhoneClaw操作手机的过程分为两步：
1. AI探索阶段：在这个阶段，AI会通过PhoneEmulationTool去不断交互式的探索界面，来完成用户的任务
2. 脚本化阶段：在这个阶段，AI已经具有足够的知识将完成任务的过程写成一个脚本，并保存起来，下次，AI会调用该脚本执行任务。

举例：
- 用户提出请求：帮助我完成企业微信打卡
- AI进入探索：不断利用PhoneEmulationTool尝试完成任务
- 脚本化：AI已经完成一次任务，具有足够的知识了，它会将当前任务保存成脚本。

下次：
- 用户提出请求，帮助我完成企业微信打卡
- AI发现已有脚本，直接执行

最终 => Token节省，效率提升

# 细节

如下是你需要考虑的一些细节

- 如何告诉大模型目前有哪些脚本？并让它决定使用？
  - 当前阶段你不应该将脚本列表作为提示词注入，而是将获取脚本列表、加载并执行脚本作为两个tool实现。
- UI自动化脚本很容易过时，比如企业微信升级了。所以，成品脚本和自动探索的一个区别就是，成品脚本必须有完善的错误发现、必须稳定、宁可执行出错，也不允许胡乱点击
- 你需要设计一个脚本列表，在主页面的侧边栏里加一个入口吧。让用户可以快速进入查看并选择、执行已有脚本，加增删改查。脚本字段：名称、简介、创建时间，点击进去可以查看脚本代码，脚本目前只读。
