此模块提供Skill的增删改查

1. 启动时异步扫描assets下的skill，按名字复制到工作目录（如果没有）
2. 允许用户对skill进行增删改查
3. skill的唯一标识是名字

提供存储的实际介质是文件/文件夹


# skill字段


根据 Anthropic 的 Skill 标准，一个 Skill 对象包含以下字段，每个字段用于控制技能的行为、匹配条件、执行环境等。下面逐一解释其用途：

| 字段 | 用途                                                                    |
|------|-----------------------------------------------------------------------|
| **name** | 技能的唯一标识符，用于在系统中引用该技能。通常采用 kebab-case（如 `code-review`）。                |
| **description** | 技能的简短描述，用于自动匹配场景（例如根据用户请求的语义判断是否调用该技能）。                               |
| **disableModelInvocation** | 是否禁止模型自动调用该技能。若为 `true`，模型不能主动触发，只能由用户显式调用。                           |
| **userInvocable** | 用户是否可以通过对话直接调用该技能（如通过 `/skill_name` 命令）。若为 `false`，只能由系统内部使用。         |
| **content** | SKILL.md 的 Markdown 内容，即技能的核心指令、工作流、示例等。这是技能定义的主体。                    |
| **skillDir** | 技能目录的路径，用于定位技能相关的资源文件（如脚本、模板、配置文件）。实现时请使用相对路径！！！                      |
| **supportingFiles** | 支持文件列表（如 `script.py`, `template.txt`）。这些文件随技能一起加载，供 `content` 中的指令使用。 |

这些字段共同定义了一个技能如何被匹配、调用、执行以及可以访问哪些资源，从而实现安全、可组合的 AI 能力扩展。
