## Why

当前聊天体验在三个核心交互上存在功能缺陷：工具调用参数未展示且未持久化、悬浮窗状态颜色未区分类型与结果、AI 消息流式拼接存在缓冲污染导致后续消息重复累加。这些问题直接影响可观测性、可读性和调试效率，需要优先修复。

## What Changes

- 修复聊天界面工具调用卡片：正确展示并记录工具调用参数（含空参/大参场景）。
- 修复悬浮窗状态配色：区分 `tool call`、`skill call`，并区分 `success`/`failure` 状态颜色。
- 修复 AI 消息流式渲染缓冲：确保每条消息独立累积，消息结束后清理 buffer，避免 A+B 累加显示。
- 补充对应状态流转与渲染一致性测试，防止回归。
- 本次实现遵循最小修改原则：仅调整聊天事件映射、相关 UI 渲染与必要持久化字段，不改动无关模块逻辑。

## Capabilities

### New Capabilities
- `chat-event-visualization`: 统一定义聊天界面对工具/技能调用事件与结果状态的可视化展示行为。

### Modified Capabilities
- `agent-execution`: 增加/修正工具调用参数传递、事件持久化与流式消息分片结束后的缓冲清理要求。

## Impact

- 受影响模块：聊天消息渲染层、工具调用事件映射层、悬浮窗状态样式层、事件持久化层。
- 数据影响：工具调用记录将包含完整参数快照（必要时做长度截断策略）。
- 测试影响：新增消息流式分片与状态颜色矩阵测试用例。
- 影响边界：不涉及模型提供商管理、工具执行引擎核心流程及其它非聊天相关模块。
